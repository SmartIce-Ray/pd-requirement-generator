# 产品研发需求

把"参考图 + 方向"变成研发能照着做的需求 PPT。四个品牌：宁桂杏 / 野百灵 / 飞花小馆 / 邦兰埔。

## 为什么有这套东西

方向由我们定、研发承接——研发缺市场、审美、用户思维这三样上游判断。所以需求 PPT 的作用：把我们已经想清楚的判断（尤其审美）说清楚、说细，让研发不走样地做出来。研发最缺审美，所以"参考与审美"那段要说得最具体。

## 怎么用（工作流）

1. 复制 `00-框架与模板/需求模板.md` 到对应品牌：`品牌/需求/需求名/需求填写.md`
2. 把参考图放进该需求的 `参考图/` 文件夹
3. 填模板 ① ② ③ 段（④ 由研发填）
4. 据此生成图片优先的 `需求PPT.pptx`
5. 把 PPT 发给研发

> 份量 / 成本 / 售价不进需求——研发出成品后按成本定价法（先做出来、按实际成本加成定价）定。

## 目录

- `00-框架与模板/` — 全品牌共用的需求模板（唯一真源，要改框架就改这里）
- `宁桂杏/` `野百灵/` `飞花小馆/` `邦兰埔/` — 每个品牌一套：
  - `品牌定位.md` — 客群 / 调性 / 价格带（填 ①② 段时参考）
  - `参考图库/` — 该品牌的灵感沉淀（还没成需求的参考图先丢这）
  - `需求/` — 每个研发需求一个子文件夹
- 示例见 `野百灵/需求/示例-夏季解腻品/`

## 参考图库归档约定

- 文件名：`场景或品类-关键词.jpg`，例如 `夏季解腻-酸梅冰饮.jpg`
- 图片来源（小红书 / 抖音 等）记在对应需求的填写稿里
