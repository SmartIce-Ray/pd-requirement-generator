## 三句话背景

现状:首页已改成干净的「日期 · N张」上传日志,点一天才展开看图;选品模式(挑图生成研发需求的入口)也得逐天点开才能勾图。变化:你指出图一多就很难挑。所以:挑图时需要"一次全铺开 + 能按品牌筛掉无关的"。

## 为什么这么设计(目标／理由／依据)

- **目标**:图多时也能快挑——减少"逐天点开找图"的翻找成本。
- **理由**:筛选只在选品模式出现,平时首页保持干净(你这两轮的明确要求 [依据 G]);自动全展开省去逐天手点。
- **依据**:筛选维度选「品牌」是你的原话;加「未打标」是因为很多图先存后标、未标的也常要选 [依据 H,待你认]。

## 交互稿(进入选品模式后)

```
[退出选品]   品牌: 全部  宁桂杏  野百灵  飞花小馆  邦兰埔  未打标   ← 只在选品模式出现的筛选栏
──────────────────────────────────────────────────
2026-06-25 · 3 张                          ← 日期组头(全部自动展开)
  ┌──┐ ┌──┐ ┌──┐
  │✓ │ │  │ │✓ │                          ← 点图 = 打勾选中
  └──┘ └──┘ └──┘
2026-06-24 · 1 张
  ┌──┐
  │  │
  └──┘
                          ┌────────────────────┐
                          │ 已选 2 · 生成研发需求 │   ← 底部选品条(现成的)
                          └────────────────────┘
```

行为:

- 点品牌(如「野百灵」)→ 只留野百灵的图,其它图和空掉的日期组隐藏。
- 点「未打标」→ 只看还没打品牌的图。
- **选中跨筛选保留**:筛野百灵选 2 张,再切飞花小馆,那 2 张仍在"购物车";筛选只换视角、不清空选中。
- 退出选品模式 → 筛选栏消失、所有天收回折叠、清空选中,回到干净日志。

> 批注：选中跨筛选不保留
